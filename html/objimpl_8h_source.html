00026 /*
00027 Functions and macros for modules that implement new object types.
00028 
00029  - PyObject_New(type, typeobj) allocates memory for a new object of the given
00030    type, and initializes part of it.  'type' must be the C structure type used
00031    to represent the object, and 'typeobj' the address of the corresponding
00032    type object.  Reference count and type pointer are filled in; the rest of
00033    the bytes of the object are *undefined*!  The resulting expression type is
00034    'type *'.  The size of the object is determined by the tp_basicsize field
00035    of the type object.
00036 
00037  - PyObject_NewVar(type, typeobj, n) is similar but allocates a variable-size
00038    object with room for n items.  In addition to the refcount and type pointer
00039    fields, this also fills in the ob_size field.
00040 
00041  - PyObject_Del(op) releases the memory allocated for an object.  It does not
00042    run a destructor -- it only frees the memory.  PyObject_Free is identical.
00043 
00044  - PyObject_Init(op, typeobj) and PyObject_InitVar(op, typeobj, n) don't
00045    allocate memory.  Instead of a 'type' parameter, they take a pointer to a
00046    new object (allocated by an arbitrary allocator), and initialize its object
00047    header fields.
00048 
00049 Note that objects created with PyObject_{New, NewVar} are allocated using the
00050 specialized Python allocator (implemented in obmalloc.c), if WITH_PYMALLOC is
00051 enabled.  In addition, a special debugging allocator is used if PYMALLOC_DEBUG
00052 is also #defined.
00053 
00054 In case a specific form of memory management is needed (for example, if you
00055 must use the platform malloc heap(s), or shared memory, or C++ local storage or
00056 operator new), you must first allocate the object with your custom allocator,
00057 then pass its pointer to PyObject_{Init, InitVar} for filling in its Python-
00058 specific fields:  reference count, type pointer, possibly others.  You should
00059 be aware that Python no control over these objects because they don't
00060 cooperate with the Python memory manager.  Such objects may not be eligible
00061 for automatic garbage collection and you have to make sure that they are
00062 released accordingly whenever their destructor gets called (cf. the specific
00063 form of memory management you're using).
00064 
00065 Unless you have specific memory management requirements, use
00066 PyObject_{New, NewVar, Del}.
00067 */
