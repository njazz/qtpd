00071 /* _PyFloat_{Pack,Unpack}{4,8}
00072  *
00073  * The struct and pickle (at least) modules need an efficient platform-
00074  * independent way to store floating-point values as byte strings.
00075  * The Pack routines produce a string from a C double, and the Unpack
00076  * routines produce a C double from such a string.  The suffix (4 or 8)
00077  * specifies the number of bytes in the string.
00078  *
00079  * On platforms that appear to use (see _PyFloat_Init()) IEEE-754 formats
00080  * these functions work by copying bits.  On other platforms, the formats the
00081  * 4- byte format is identical to the IEEE-754 single precision format, and
00082  * the 8-byte format to the IEEE-754 double precision format, although the
00083  * packing of INFs and NaNs (if such things exist on the platform) isn't
00084  * handled correctly, and attempting to unpack a string containing an IEEE
00085  * INF or NaN will raise an exception.
00086  *
00087  * On non-IEEE platforms with more precision, or larger dynamic range, than
00088  * 754 supports, not all values can be packed; on non-IEEE platforms with less
00089  * precision, or smaller dynamic range, not all values can be unpacked.  What
00090  * happens in such cases is partly accidental (alas).
00091  */
00092

00094  * argument, true if you want the string in little-endian format (exponent
00095  * last, at p+3 or p+7), false if you want big-endian format (exponent
00096  * first, at p).
00097  * Return value:  0 if all is OK, -1 if error (and an exception is
