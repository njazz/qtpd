00016    Each interface exports both functions and macros.  Extension modules should
00017    use the functions, to ensure binary compatibility across Python versions.
00018    Because the Python implementation is free to change internal details, and
00019    the macros may (or may not) expose details for speed, if you do use the
00020    macros you must recompile your extensions with each Python release.

00022    Never mix calls to PyMem_ with calls to the platform malloc/realloc/
00023    calloc/free.  For example, on Windows different DLLs may end up using
00024    different heaps, and if you use PyMem_Malloc you'll get the memory from the
00025    heap used by the Python DLL; it could be a disaster if you free()'ed that

00027    can return the memory to the proper heap.  As another example, in

00029    memory functions in special debugging wrappers that add additional
00030    debugging info to dynamic memory blocks.  The system routines have no idea
