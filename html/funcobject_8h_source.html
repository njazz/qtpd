00010 /* Function objects and code objects should not be confused with each other:
00011  *
00012  * Function objects are created by the execution of the 'def' statement.
00013  * They reference a code object in their func_code attribute, which is a
00014  * purely syntactic object, i.e. nothing more than a compiled version of some
00015  * source code lines.  There is one code object per source code "fragment",
00016  * but each code object can be referenced by zero or many function objects
00017  * depending only on how many times the 'def' statement in the source was
00018  * executed so far.
00019  */
00020
