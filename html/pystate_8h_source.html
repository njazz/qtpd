00139 /* Ensure that the current thread is ready to call the Python
00140    C API, regardless of the current state of Python, or of its
00141    thread lock.  This may be called as many times as desired
00142    by a thread so long as each call is matched with a call to
00143    PyGILState_Release().  In general, other thread-state APIs may
00144    be used between _Ensure() and _Release() calls, so long as the
00145    thread-state is restored to its previous state before the Release().
00146    For example, normal use of the Py_BEGIN_ALLOW_THREADS/
00147    Py_END_ALLOW_THREADS macros are acceptable.
00148 
00149    The return value is an opaque "handle" to the thread state when
00150    PyGILState_Ensure() was called, and must be passed to
00151    PyGILState_Release() to ensure Python is left in the same state. Even
00152    though recursive calls are allowed, these handles can *not* be shared -
00153    each unique call to PyGILState_Ensure must save the handle for its
00154    call to PyGILState_Release.
00155 
00156    When the function returns, the current thread will hold the GIL.
00157 
00158    Failure is a fatal error.
00159 */
00160 PyAPI_FUNC(PyGILState_STATE) PyGILState_Ensure(void);
