00005 /* this file defines the structure for "glists" and related structures and
00006 functions.  "Glists" and "canvases" and "graphs" used to be different
00007 structures until being unified in version 0.35.
00008 
00009 A glist occupies its own window if the "gl_havewindow" flag is set.  Its
00010 appearance on its "parent", also called "owner", (if it has one) is as a graph
00011 if "gl_isgraph" is set, and otherwise as a text box.
00012 
00013 A glist is "root" if it has no owner, i.e., a document window.  In this
00014 case "gl_havewindow" is always set.
00015 
00016 We maintain a list of root windows, so that we can traverse the whole
00017 collection of everything in a Pd process.
00018 
00019 If a glist has a window it may still not be "mapped."  Miniaturized
00020 windows aren't mapped, for example, but a window is also not mapped
00021 immediately upon creation.  In either case gl_havewindow is true but
00022 gl_mapped is false.
00023 
00024 Closing a non-root window makes it invisible; closing a root destroys it.
00025 
00026 A glist that's just a text object on its parent is always "toplevel."  An
00027 embedded glist can switch back and forth to appear as a toplevel by double-
00028 clicking on it.  Single-clicking a text box makes the toplevel become visible
00029 and raises the window it's in.
00030 
00031 If a glist shows up as a graph on its parent, the graph is blanked while the
00032 glist has its own window, even if miniaturized.
00033 
00034 */
00035
